
             ---===[ Qubes Security Bulletin 111 ]===---

                             2026-03-26

                     xfce4-screensaver login bypass

User action
------------

Continue to update normally [1] in order to receive the security updates
described in the "Patching" section below. No other user action is
required in response to this QSB.

Summary
--------

When activating the screensaver login prompt on a locked screen, there
is a very short time when keyboard input is not intercepted by
xfce4-screensaver and is instead sent to the application that was active
before locking, bypassing the login prompt.

Impact
-------

The time window normally is too short to do real damage manually. But if
an attacker is able to automatically generate input events the attacker
can try to deactiveate the screensaver by typing a command quickly
enough.

Affected systems
-----------------

Qubes 4.3 with xfce4-screensaver as the screensaver (the default option
with the default Xfce desktop environment) is affected. Other Qubes
versions and other screensavers are not affected.

If the system is configured to allow USB keyboard input without a
confirmation prompt (off by default) it's easy for an attacker to
quickly generate input events by using an USB device that emulates an
keyboard.

Systems that otherwise expose a keyboard input interface are affected
even when USB keyboard input without configuration is not allowed. The
most common, but nowadays overall still uncommon, option is PS/2 input.

An attacker could also access internal interfaces through physical
modifications. For example the connection of an integrated keyboard of a
laptop. This is more complicated than the above options and requires
physical access to an extend that other attacks that are out of scope to
Qubes' security model are possible too.

The time window in which the input events are wrongly delivered seems to
heavily vary between systems (between non being able to get even one key
event through, to users being able to manually type a full password).
The exact circumstances are not understood yet, but it might be related
to the interaction of the graphic stack with the hardware installed in a
system.

Patching
---------

The following packages contain security updates that address the
vulnerabilities described in this bulletin:

  For Qubes 4.3, in dom0:
  - xfce4-screensaver 4.18.4-5

For GUI domain users, the relevant template will get the update
through appropriate distribution repositories (Fedora, Debian, etc).

These packages will migrate from the security-testing repository to the
current (stable) repository over the next two weeks after being tested
by the community. [2] Once available, the packages should be installed
via the Qubes Update tool or its command-line equivalents. [1]

Dom0 user session must be restarted afterward in order for the updates
to take effect.
